Verantwoording gebruike code

Deze website gebruikt niet het standaard '960px grid' dat veel leerlingen hebben gebruikt. In plaats daarvan, gebruik ik een simpele elementklasse 'maxwidth', wat als 'row' dient, met daarin vooral op 'flexbox' gebaseerde uitlijning. De 'maxwidth' klasse bepaald de maximum breedte van de content en schaalt dynamisch mee met de breedte van de pagina.

Mijn portfoliopagina bevat een voorbeeld van mijn eigen systeem: Alle portfolio-items krijgen elk een breedte van 33%, met een padding die de groote van het element er binnen bepaalt. Om dit systeem responsive te maken, wordt met behulp van de css 'nth-child' selector en media queries de padding links of rechts op 0 gezet, zodat de randen aansluiten op de maximum breedte van de pagina.

Mijn docent, C.F.S. Bot, vond mijn eigen systeem prima. Tijdens het overleg met haar vroeg ik ook of het toegestaan was om andere, complexere systemen te gebruiken zoals node.js om deze website op te bouwen. Ik mocht in principe alles gebruiken, mits ik het kon toelichten en het niet direct van internet geplukt was.

Template systeem

Naast mijn eigen gridsysteem heb ik dus ook een template systeem gebruikt, wat draait op node.js en gulp.js. Het javascript gebaseerde node.js is gemaakt om open-source 'packages' in je project te laden die verschillende taken voor je kunnen uitvoeren.

Ik heb gekozen om het pakket 'gulp' te installeren: een build tool waarmee je het aanpassen van bestanden kan automatiseren, wat kan worden uitgebreid met andere node-gebaseerde pakketten. Binnen het bestand 'Gulpfile.js' geeft je aan welke pakketten moeten worden geladen en op welke volgorde de taken moeten worden uitgevoerd.

Voor dit project heb ik 'gulp-nunjucks-render' gebruikt om .nunjucks templates in mijn html te kunnen laden. Voordat de templatebestanden zijn ingeladen, leest gulp eerst nog het bestand 'data.json' uit met behulp van het pakket gulp-data. Hierin heb ik alle hobby's en portfolio items gezet, zodat er met behulp van een 'loop' een aantal elementen worden gerenderd. Nadat de html is gebouwd, worden met behulp van het pakket 'gulp-html-beautify' alle tab-indents netjes gemaakt.

Dus, wanneer via de commandline het commando 'gulp' wordt aangeroepen, pakt het programma al mijn html-bestanden uit de /src/ map, templates uit /src/partials/ en data uit /src/data/data.json. De bestanden worden via het ramgeheugen doorgegeven aan elke taak via '.pipe()', waarna de afgemaakte bestanden in de /app/ map worden geplaatst.

Het hele project kan worden terug gevonden op mijn persoonlijke Github pagina.
