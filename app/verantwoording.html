Verantwoording

De gemaakte ontwerpkeuzes van deze website zijn gebaseerd op achterliggende theorieën: de heuristieken van Nielsen en de algemene gestaltprincipes. De elementen uit deze theorieën worden hieronder toegelicht en geïllustreerd met behulp van voorbeelden die te zien zijn op deze website.

Heuristieken

Match between system and the real world

Een website zou overeenkomen met dingen uit het echte leven en niet voelen als output van een machine. Zo heb ik alle teksten op de website geschreven in 'gewone taal', met als doel om dicht bij de gebruiker te blijven.

User control and freedom

Een gebruiker zou altijd de mogelijkheid moeten hebben om terug te gaan naar de vorige status van een programma. Dit komt in deze website terug in de portfolio detailpagina's: er wordt een knop getoond waarmee de gebruiker terug kan navigeren naar het overzicht.

Consistency and standards

De website volgt een aantal principes die op de meeste websites worden toegepast, zodat de gebruiker een vertrouwd gevoel krijgt. Zo staat bijvoorbeeld het logo van de website in de linkerbovenhoek en de navigatiebalk in de rechterbovenhoek.

Recognition rather than recall

Het gebruik van bekende elementen in een website is erg belangrijk voor een gestroomlijnde ervaring tijdens het surfen. Zo heb ik bijvoorbeeld het hamburgermenu in de mobiele weergave niet een compleet ander uiterlijk gegeven, maar ben bij de 'goede oude' drie lijntjes gebleven die elke gebruiker zal herkennen.

Aesthetic and minimalist design

Om een overlading van informatie te voorkomen, is het van belang dat de website op de meest minimale manier wordt weergegeven. Ik heb er dus voor gekozen om de pagina's op de meest simpele manier vorm te geven. Zo heeft deze pagina bijvoorbeeld geen afbeeldingen of te lange teksten.

Gestaltprincipes

Figure-ground

Wanneer een voor- en achtergrond geen duidelijk onderscheid hebben, moeten gebruikers zelf een keuze maken wat ze als voorgrond zien. Om een duidelijk contrast te maken tussen voor- en achtergrond heb ik bijvoorbeeld op de homepagina een volledig gevulde achtergrondkleur gebruikt, met daarop een wit logo. In dit geval lijkt het alsof het logo de voorgrond is.

Proximity

Elementen worden gegroepeerd gezien wanneer ze dicht bij elkaar staan ten opzichte van de rest. Zo worden alle bij elkaar horende stukken tekst onderverdeeld, waarbij de tekst vrij dicht bij zijn bijbehorende kopje staat. Ook staan bijvoorbeeld de navigatie-items bij elkaar en worden niet verspreid over de volledige breedte van de pagina.

Similarity

Dingen die bij elkaar horen zouden er hetzelfde uit moeten zien. Zo zien alle knoppen op mijn website er hetzelfde uit, maar ook bijvoorbeeld de items in het portfolio en de hobby's op de hobby's pagina.

Continuity

Elementen worden gegroepeerd op basis van hun uitlijning ten opzichte van elkaar, waarbij het meest logische pad een groep vormt. Zo heeft de sitemap bijvoorbeeld een hiërarchische structuur, waardoor het duidelijk is welke links bij elkaar horen en onder welke pagina ze zich bevinden.

Closure

Door elementen een duidelijk kader te geven worden ze gegroepeerd. Op de hobby's pagina zie je bijvoorbeeld dat het icoon bij de tekst aan zijn zijde hoort, en niet bij de tekst er onder: de achtergrondkleur vormt een kader.

Verantwoording gebruikte code

Deze website gebruikt niet het standaard '960px grid' dat veel leerlingen hebben gebruikt. In plaats daarvan, gebruik ik een simpele element klasse 'maxwidth', wat als 'row' dient, met daarin vooral op 'flexbox' gebaseerde uitlijning. De 'maxwidth' klasse bepaald de maximum breedte van de content en schaalt dynamisch mee met de breedte van de pagina.

Mijn portfolio pagina bevat een voorbeeld van mijn eigen systeem: Alle portfolio-items krijgen een breedte van 33%, met een padding die de grootte van het element er binnen bepaalt. Om dit systeem responsive te maken, wordt met behulp van de css 'nth-child' selector en media queries de padding links of rechts op 0 gezet, zodat de randen aansluiten op de maximum breedte van de pagina.

Mijn docent, C.F.S. Bot, vond mijn eigen systeem prima. Tijdens het overleg met haar vroeg ik ook of het toegestaan was om andere, complexere systemen te gebruiken zoals node.js om deze website op te bouwen. Ik mocht in principe alles gebruiken, mits ik het kon toelichten en het niet direct van internet geplukt was.

Template systeem

Naast mijn eigen gridsysteem heb ik naar aanleiding van het gesprek met mijn docent C.F.S. Bot ook een template systeem gebruikt, wat draait op node.js en gulp.js. Het javascript gebaseerde node.js is gemaakt om open-source 'packages' in je project te laden die verschillende taken voor je kunnen uitvoeren.

Ik heb gekozen om het pakket 'gulp' te installeren: een build tool waarmee je het aanpassen van bestanden kan automatiseren, wat kan worden uitgebreid met andere node-gebaseerde pakketten. Binnen het bestand 'Gulpfile.js' geeft je aan welke pakketten moeten worden geladen en op welke volgorde de taken moeten worden uitgevoerd.

Voor dit project heb ik 'gulp-nunjucks-render' gebruikt om .nunjucks templates in mijn html te kunnen laden. Voordat de template bestanden zijn ingeladen, leest gulp eerst nog het bestand 'data.json' uit met behulp van het pakket gulp-data. Hierin heb ik alle hobby's en portfolio items gezet, zodat er met behulp van een 'loop' een aantal elementen worden gerenderd. Nadat de html is gebouwd, worden met behulp van het pakket 'gulp-html-beautify' alle tab-indents netjes gemaakt.

Dus, wanneer via de commandline het commando 'gulp' wordt aangeroepen, pakt het programma al mijn html-bestanden uit de /src/ map, templates uit /src/partials/ en data uit /src/data/data.json. De bestanden worden via het ram geheugen doorgegeven aan elke taak via '.pipe()', waarna de uiteindelijke bestanden in de /app/ map worden geplaatst.

Het hele project kan worden teruggevonden op mijn persoonlijke Github pagina.
